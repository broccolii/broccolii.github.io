May 7, 2016 | Hits

作为部门报销参加 SwiftCon 费用的条件，要做一次关于 Swift 的技术分享。现在想想去参加 SwiftCon，整个人尴尬症都犯了（原因详见）。不过好在之前接触过一段时间的 Swift，不至于没东西可以讲，趁这次机会学习一下 Alamofire 的源码。本文中使用的 Alamofire 版本是 3.3.1。

Usage

我们先看一下官方 GET 请求的姿势：

Alamofire 调用 request 方法（剧透一下：这里不是调用，而是一个命名空间）传入 HTTP method 、API 、parameters。再链式调用 responseJSON 方法通过 尾随闭包 回调 response 的结果。

目录结构

我们先 clone 一份代码看一下目录结构。目录结构如下，Alamofire 的实现文件并不多，一方面是因为这是一个全新的库没有历史的抱负，另一方面归功于 Swift 一些特有的语法特性。

一共三部分内容：

  • Alamofire 是一个命名空间，在 Alamofire.swift 中定义了一些外部使用的 API。
  • Core 中是一些核心组件：Manager 封装了 Request 操作，Response 和其包涵的数据结果 Result，还有一些异常定义和参数拼接方法。
  • Features 主要是对 Core 中几个组件的 extension

Alamofire.swift

看文件结构发现其实只做了三件事情：

  1. 定义了两个协议 URLStringConvertible 和 URLRequestConvertible 并拓展了几个原生类
  2. 写了一个创建 NSMutableURLRequest 的 Convenience method
  3. 封装了 Manager 的三类方法：request、upload、download

来说一下两个协议 URLStringConvertible 和 URLRequestConvertible，两个协议的后缀都是 -Convertible。

系统协议后缀一般常使用 -Type、-able、-Convertible 这三种。

我们可以看一下 Swift 的 Array 和 Dictionary 的源码中默认实现的协议：


Array



Dictionary


顾名思义: -Type 指的是一种类型，-able指的是能够实现怎么样一种行为，-Convertible指的是能进行类型转换。URLStringConvertible 便是将不同类型转换成 String 类型。

Response.swift 和 Result.swift

前文中已经发出一个了 Request， 我们需要用 Response 来接收数据，而请求的结果则是使用 Result 表示。
我们先看一下 Result 是怎么定义的：

Result 是一个枚举，但是 Value 和 Error: ErrorType 又是什么呢。如果换成 enum Result<T, U: ErrorType> 是不是一下子明白是泛型了。Swift 中的枚举还能定义属性方法等。建议查看文档中的泛型和枚举两章。

Response 和 Result 类似是一个泛型结构体，可以存储一些请求数据、响应数据等。
