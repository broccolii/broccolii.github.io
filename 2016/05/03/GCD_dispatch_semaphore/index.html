今天在写 FMDB 时，遇到了一个异步调用中等待的问题需要用到 dispatch_semaphore。碰巧之前土豆面试我时，问到 dispatch_barrier_async 我搞混成了前者。所以写篇 blog 整理一下 dispatch_semaphore 和 dispatch_barrier_async 两者的区别。

解决办法：使用 dispatch_semaphore

dispatch_semaphore 简单理解就是信号量。了解过 RAC 的同学应该都不会陌生。先上解决办法

  • dispatch_semaphore_t dispatch_semaphore_create(long value);
  • long dispatch_semaphore_signal(dispatch_semaphore_t dsema);
  • long dispatch_semaphore_wait(dispatch_semaphore_t dsema, dispatch_time_t timeout);

简单的来说：

  • dispatch_semaphore_create 创建一个信号，传入是 value 代表可用资源数。当资源数等于零时进入等待状态，大于零表示有资源则线程可以进行下去。*
  • dispatch_semaphore_wait 表示等待状态设置超时时间（时间单位是十亿分之一秒）。
  • dispatch_semaphore_signal 传入之前的 dsema 信号量资源数加一，那么线程得以继续运行下去。

dispatch_barrier_async

barrier 的中文意思是： 障碍物。dispatch_barrier_async可以在并发队列中创建一个同步点。即在 dispatch_barrier_async 之前的异步操作执行完才会进入 barrier block，在执行 barrier block 队列中只允许存在此操作。如下图所示：
