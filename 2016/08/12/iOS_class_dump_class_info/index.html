Aug 12, 2016 | Hits

前段时间盘古出了 iOS 9.3.3 的越狱，尝试把手机越狱学习一下 iOS 安全方面的新知识。简单尝试了一下网上比较多教程的 class-dump 分析 App 类信息。因为实际操作的时候遇到了好一些问题，随记一篇以备日后查阅。

环境准备

OpenSSH
默认是越狱环境下，首先是在 Cydia 中下载安装 OpenSSH。

OpenSSH 默认的密码是 alpine，为了安全起见，我们点击上图中的 Root Password How-To，有详细的教程教我们修改密码，在此就不过多介绍了。

BigBoss Recommendation tools
接下去安装 BigBoss Recommendation tools，这个包中包含了很多工具啦~

SSH
准备工作差不多了，现在我们通过 SSH 登录进这个越狱设备。确保你的电脑和越狱设备连接在同一个网络下。

可以看到我们现在的 IP 是 192.168.1.42，我们使用刚刚修改好的密码登录 root 用户。

class-dump-z
我们在官方地址下载，最新的 class-dump-z (class-dump-z_0.2a)。你可能看到其他教程里有写到 class-dump 或者 class-dump-x。这三者的区别具体请看lzwjava的blog这里不多做描述。

网上的 blog 大多数都比较早，好多讲到将 class-dump-z 放到 iOS 设备上使用。其实 class-dump-z 只支持 32bit的设备。具体请看文档

Instead of a generic-purpose class dumper, class-dump-z was written with iPhoneOS development in mind. Therefore, the following features will probably never be implemented: 64-bit support. (Unless there is a 4 GiB RAM iPhone.) Objective-C 1.0 ABI. In general, if you want to dump Mac OS X, you should use the original class-dump. But if you’re dumping iPhoneOS binaries, class-dump-z is definitely the better choice.

导出 WeChat 的类信息

我们需要使用 Clutch 用来破壳。原因是从 App Store 下载的 App 都是经过签名加密的，可执行文件被加上了一层“壳”。

我们从 GitHub 上下载最新发布的 Clutch，解压后用 sftp 或者 scp 上传到越狱手机的 /usr/bin 目录下。
