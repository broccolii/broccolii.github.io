macOS 执行定时脚本

Aug 11, 2016 | Hits

有时候我们要在 Mac 上添加一些定时任务，我们可以通过 crontab 或者 launchctl 添加定时任务。

crontab 命令

crontab 是 Linux 下的定时任务命令：

基本参数
crontab [-u username] [-l|-e|-r]

-u ：只有 root 才能进行这个任务，也就是帮其他使用者建立/移除 crontab 工作排程；
-e ：编辑 crontab 的工作內容
-l ：查看 crontab 的工作內容
-r ：移除所有的 crontab 的工作內容，若仅仅移除一项，请用 -e 去编辑。

基础格式

(星号) 代表任意时刻
,(逗号) eg:3,6 代表在每个小时的3分以及6分执行
-(减号) 代表时间范围。eg:06 8-12 代表在8点到12点每个小时的第6分钟执行
/n(除n) n代表数字，每隔n分钟执行一次。eg:/30 command代表每半个小时执行一次

如果你用 crontab -e 编辑完之后会提示

crontab: “/usr/bin/vi” exited with status 1

那么请使用 EDITOR=vim crontab -e

我们可以测试一下，使用 crontab -e 在文件中添加 * * * * * say hello，每分钟电脑就会播放 hello。

luanchctl 定时任务

luanchctl 通过 plist 配置文件的方式来实现定时任务的。比 crontab 好用的地方在于可以间隔一秒出发定时任务。

plist 配置路径：
/Library/LaunchDaemons 和 /Library/LaunchAgents。
其区别是第二个路径中的脚本只有在用户登陆系统后才会被执行。
第一个只要系统启动了，哪怕用户不登陆系统也会被执行。

可以通过两种方式来设置脚本的执行时间：

StartInterval：它指定脚本每间隔多长时间（单位：秒）执行一次
StartCalendarInterval：它可以指定脚本在多少分钟、小时、天、星期几、月时间上执行，类似crontab

这样我们的定时任务就已经配置好了，那么怎么启动起来呢？

我们可以使用一下几个命令操作我们的定时任务

要让任务生效，必须先 load 命令加载这个 plist
如果任务被修改了，那么必须先 unload，然后再次 load
start 立即执行任务，可以用于测试定时任务是否配置成功
执行 start 和 unload 前，任务必须先 load
stop 可以停止任务
list 显示所有任务
